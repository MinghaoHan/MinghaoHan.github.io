Hexo+Next 博客升级日志

发表于 2022-10-26 更新于 2022-11-01 阅读次数： Waline： 本文字数： 13k 阅读时长 ≈ 12 分钟

前端小菜鸡，到处搬运大佬的博客搭建教程，最后有了自己的博客。把教程都整理了一下。

写在前面

先给自己搭一个，本文讲的是NexT主题配置以及页面的样式优化，参考了许多大佬的文章以及配置文件。在此记录一下做过哪些改动，方便以后自定义，and帮女朋友搭博客。

基本定义

在Hexo中自定义主要靠两个文件，./_config.yml(总配置文件) 和 ./themes/next/_.config.yml(主题配置文件)。总配文件匹配的起始目录是source/。
在Hexo 8.13.1中，各种自定义文件都放在./source/_data下；各种博文都在./source/_posts下。

Hexo

根据官网，安装hexo。首先是基础软件 nodejs 以及 git，这两个都是常用基础软件所以就不介绍细节了，官网直接安装即可。我们要使用 npm 这个 nodejs 库管理软件。

由于默认源在国外经常连接超时，所以首先需要修改成国内的镜像源，这里是改成了淘宝的镜像源。

npm config set registry https://registry.npm.taobao.org

基本命令

hexo init：生成新博客文件夹
hexo new (page) "xxx"：生成新文章
hexo clean ：清cache
hexo generate：生成( hexo g )
hexo deploy：部署( hexo d)

安装NexT主题

hexo有上百种主题，个人使用的是NexT。安装NexT最新release版本：

git clone --branch v8.11.1 https://github.com/next-theme/hexo-theme-next themes/next

然后修改总配文件中：

theme: next

Hexo配置

修改总配文件的一些参数，可以参考官方文档。

修改博客基本信息

title: <博客主题>
subtitle: <相当于个性签名>
description: <相当于个性标签>
keywords: <关键词>
author: <博主>
language: <语言（zh-CN）>
timezone: <时区>

同步生成文件夹

将每篇博客对应生成一个文件夹目录（方便插入图片等附件）：

post_asset_folder: true

md 文件中加入 就可以。

部署方式

设置完此项后，可以用hexo d一键部署。

deploy:
  type: git
  repo: <github token>@<仓库git地址>
  branch: master

搜索功能

NexT 主题自带了一个搜索功能 Local Search，即在编译文件时本地生成一个数据库，放在网站根目录下，用户借助此数据库进行搜索查询。先安装：

npm install hexo-generator-searchdb --save

修改总配文件：

search:
  path: search.xml
  field: post
  format: html
  limit: 10000

修改主配文件：

local_search:
  enable: true

阅读时间统计

给博客文章添加一下字数以及阅读时间统计。安装：

npm install hexo-symbols-count-time

总配文件中加入(awl和wpm参数按照中英混合计算，纯中文为3、200，自行估算)：

symbols_count_time:
  symbols: true
  time: true
  total_symbols: true
  total_time: true
  exclude_codeblock: false
  awl: 4
  wpm: 275
  suffix: "mins."

添加简历

新建about页面，Markdown中插入PDF。简历PDF制作，可以选择超级简历。在md文件中：

<object height="700" width="950" border="0" data="path/my.pdf" type="application/pdf">
    <embed src="path/my.pdf" type="application/pdf" width=800 height=800 />
</object>

或者

<iframe src="/pdf/my.pdf" style="width:700px; height:950px;" frameborder="0"></iframe>

或者在新窗口打开：

<object data="http://yoursite.com/the.pdf" type="application/pdf" width="700px" height="700px">
    <embed src="http://yoursite.com/the.pdf">
        <p>This browser does not support PDFs. Please download the PDF to view it: <a href="http://yoursite.com/the.pdf">Download PDF</a>.</p>
    </embed>
</object>

不渲染Markdown文件

在总配文件中，skip_render参数后加入具体文件。添加多个文件如下：

skip_render: README.md
  - `test1/*.html`
  - `test2/**`

优化

动态背景

使背景不那么单调，加入了一个几乎不影响性能的背景动画。 在 source/_data 目录 (若不存在则新建一个) 下新建文件 footer.njk，内容：

<script color="0,0,255" opacity="0.5" zIndex="-1" count="99" src="https://lib.baomitu.com/canvas-nest.js/1.0.1/canvas-nest.js"></script>

可以修改数量，透明度，颜色。
在主配文件中找到 custom_file_path 一项将 footer 一行取消注释：

custom_file_path:
  footer: source/_data/footer.njk

添加评论系统

本博客使用的是 Waline 评论系统，是基于 LeanCloud 存储的系统。使用的是最简单、零成本的部署方式。

Step1. 注册客户端。注册一个LeanCloud国际版账号。国内版需要备案的域名，国际版有公用的。如果显示国际版不支持中国大陆，请忽略它。

Step2. 注册完成后，登录，然后找到创建应用。填写你的应用名称,名称可以自己定义，然后，下面选择开发版 点击创建。点击应用进入设置。

Step3. 点击应用凭证，取得我们 AppKey 、App id 、以及 MasterKey。

Step4. 安装服务端。Waline 支持多种服务端，为了最简便上手，我们使用在 Vercl 上安装服务端。首先，点击 一键部署。需要注册一个账号，支持使用 Github 账号直接登录。登录后重新点进来，点击 Create。输入好点击Create后，等待下面 Deploy 构建完成，点击 Go to Dashboard。

Step5. 找到 Settings => Environment Variables，配置环境变量。三个环境变量对应先前的应用凭证：

LeanCloud Vercel Environment
AppID LEAN_ID
AppKey LEAN_KEY
MasterKey LEAN_MASTER_KEY
  • 如果你使用 LeanCloud 国内版，请额外配置 LEAN_SERVER 环境变量，值为你绑定好的域名。

Step6. 为了使环境变量生效，需要重新构建一次。在上方找到 Deployments ，选择第一个右边的三个点，点击 Redeploy 。等待其构建结束，然后记住 DOMAINS 中的域名地址。

Step7. 配置。安装：

npm install @waline/hexo-next

在主配中加入

# Waline
# For more information: https://waline.js.org, https://github.com/walinejs/waline
waline:
  enable: true #是否开启

  # Warning: 不要同时启用 `waline.visitor` 以及 `leancloud_visitors`.
  visitor: false # 文章阅读统计
  comment_count: true # 如果为 false , 评论数量只会在当前评论页面显示, 主页则不显示
  requiredFields: [] # 设置用户评论时必填的信息，[nick,mail]: [nick] | [nick, mail]
  libUrl: # Set custom library cdn url

登陆服务端

Waline 有服务端，支持评论管理。我们需要注册一个账号作为管理员。服务端。

找到评论框，点击 登录 按钮，会弹出一个窗口，找到用户注册，默认第一个注册的用户为管理员，所以部署好一定要记得及时注册。

注册好，登录之后即可进入评论管理的后台，可以对评论进行管理。

相册

Step1. 安装Picgo，部署github仓库存放图片。也可以使用七牛云，超过10G会收费。

Step2. hexo n page "gallery"，在主配文件夹menu添加gallery，在next/language/zh-CN.yml menu添加gallery: 相册。

Step3. 图片在Picgo上传后，复制链接到gallery的md文件中。

部署在Github

  • 优点：免费薅羊毛，不限流量，支持静态脚本
  • 缺点：国内访问不稳定，图片可能加载不出来
  • 对Github不熟悉的朋友，可以参考博客下的Github教程。
